# Contributing

Thanks for your interest in contributing!

## How to Contribute
- Open an issue to discuss bugs or enhancements.
- Submit a PR with clear context and a focused scope.
- Keep changes aligned with the goals: cost-aware, reliable PR review automation.

## Development Guidelines
- Prefer small, isolated changes.
- Keep documentation concise and practical.
- Avoid introducing vendor-specific dependencies when possible.

## Pull Request Checklist
- [ ] Clear description of the change
- [ ] Tests or reasoning for why tests are not needed
- [ ] Documentation updates (if behavior changes)

This project follows `CODE_OF_CONDUCT.md`.
